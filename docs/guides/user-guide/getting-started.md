# {{ $frontmatter.title }}

На этой странице описано, как вы можете начать использовать наш редактор StackBlitz.

Если вы хотите сразу встроить свой проект на свою страницу, проверьте страницу руководства [embeds](/guides/integration/embedding) и [SDK embeds](/platform/api/javascript-sdk). Если вы хотите узнать, как импортировать проект из GitHub, ознакомьтесь со страницей [импорт проектов](/guides/user-guide/importing-projects).

## Выбор стартового проекта

Чтобы начать кодирование, перейдите на [домашнюю страницу StackBlitz](https://stackblitz.com/). Если вы не вошли в систему, прокрутите страницу вниз, чтобы найти список стартовых проектов на выбор:

![Project Starter Dashboard](./assets/project-starters-public.png)

Если вы вошли в систему, вы должны увидеть свою личную приборную панель с очень похожим списком стартовых проектов:

![Project Starter Dashboard](./assets/project-starters.png)

Вы можете изучить различные категории (например, "Популярные", "Frontend", "Fullstack"), чтобы найти технологический стек, который вы хотели бы попробовать.

Некоторые фреймворки также предоставляют короткие URL, которые можно использовать для создания новых проектов StackBlitz. Посмотрите [список ".новых доменов" здесь](/guides/user-guide/starter-projects#new-domains).

## Импортирование кода

Нет стартового проекта для вашего любимого фреймворка? Или, может быть, вы хотите импортировать свой собственный код?

Не волнуйтесь! В StackBlitz вы можете [импортировать проекты](/guides/user-guide/importing-projects) из собственных файлов или из репозиториев GitHub.

## Использование редактора StackBlitz

После того, как вы выбрали или загрузили проект, вы должны увидеть редактор StackBlitz, который автоматически компилирует код проекта, чтобы показать результат на панели предварительного просмотра справа. Теперь внесите некоторые изменения в код и смотрите обновления в режиме реального времени в Preview!

![The StackBlitz editor showing the React TypeScript starter project](./assets/getting-started-editor.png)

Заинтригованы всем тем, что видите на экране? Ознакомьтесь с [различными частями редактора](/guides/user-guide/ide-whats-on-your-screen).

:::tip Совет: войдите в систему, чтобы сохранить свою работу
Если вы еще не вошли в систему, мы рекомендуем войти под учетной записью GitHub, чтобы убедиться, что вы сможете сохранить свои изменения и продолжить работу позже.
:::

## Использование StackBlitz на вашем сайте

Если вы пишете блог, поддерживаете сайт документации или проект с открытым исходным кодом, вы можете использовать StackBlitz для:

1. [Embed examples and demos](/guides/integration/embedding) на вашей странице или в сообщениях вашего блога.
2. [Добавьте кнопки "Открыть в StackBlitz"](/guides/integration/open-from-github) в ваши README и документации.
3. [Создание пользовательских проектов на лету] (/guides/integration/create-with-sdk) с помощью нашего JavaScript SDK.
4. [Используйте репродукции StackBlitz в GitHub](/guides/integration/bug-reproductions) шаблоны вопросов об ошибках!

Счастливого кодирования!
